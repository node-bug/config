# Security Policy

## Supported Versions

The following versions of @nodebug/config are currently being supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 2.1.x   | :white_check_mark: |
| 2.0.x   | :x:                |
| < 2.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability within this project, please send an email to nodebug@outlook.com. All security vulnerabilities will be promptly addressed.

Please include the following information in your report:

- A clear description of the vulnerability
- Steps to reproduce the vulnerability
- The affected version(s)
- Any potential impact of the vulnerability
- Suggested fix or mitigation

We will respond to your report within 48 hours and keep you updated on our progress toward resolving the issue.

## Security Best Practices

- Keep your dependencies up to date
- Review configuration files for sensitive information
- Use environment variables for sensitive data
- Regularly audit your application for security issues

## Security Updates

Security updates will be released as needed and will be clearly documented in the release notes.
